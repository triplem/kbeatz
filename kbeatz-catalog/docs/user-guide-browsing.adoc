== Overview

kbeatz is a music collection management platform for FLAC libraries.
The browser UI at `http://localhost:3005` lets you browse your entire album collection
at a glance, narrow it with filters and free-text search, and sort by album artist or composer.

Album browsing uses *server-side pagination and filtering* via the catalog API.
The UI requests one page of albums at a time and sends active filters as query parameters.
Multi-value filters (multiple genres, artists, or composers selected simultaneously) fall back
to client-side evaluation on the current page until a dedicated multi-value server endpoint is added.

== Album Grid

When the page loads, kbeatz fetches the first page of albums from the catalog service and renders them
as a responsive grid of cards.

Each *album card* shows:

* *Cover art* - the front cover image served by the catalog.
  If no cover is available, a placeholder icon is shown instead.
* *Title* - the album (ALBUM Vorbis tag).
* *Primary attribution* - see <<_classical_music_display>>.
* *Year* - the DATE Vorbis tag (year or full ISO date).
* *Genre* chip - the GENRE Vorbis tag.

The grid uses CSS auto-fill so cards wrap naturally from a single column on mobile to
multiple columns on wider screens.

=== Scan Progress Banner

If a library scan is in progress when you open the UI, a banner appears at the top:

....
Scanning: 342 / 2000 albums
....

The count updates every 2 seconds.
The banner disappears automatically when the scan completes.
If the scan fails, the banner displays the error message from the server.

== Filtering the Collection

The *filter panel* appears on the left side of the grid.
Single-value filters (one genre, artist, or composer selected) are applied server-side;
the catalog returns only matching albums.
Selecting multiple values in the same filter category applies those filters client-side
on the current page as an interim measure until a multi-value server endpoint is available.

=== Genre Filter

Check one or more genre boxes to show only albums matching those genres.
Multiple genres use *OR* logic: checking "Jazz" and "Blues" shows albums in either genre.

image::placeholder.png[Screenshot placeholder - genre filter checkboxes]

=== Artist Filter

Check one or more artist boxes to restrict the grid to albums by those album artists.
Multiple artists use OR logic.

=== Composer Filter

Check one or more composer boxes to show only albums with those composers set.
Albums without a COMPOSER tag are excluded when any composer filter is active.

=== Combining Filters

Different filter axes combine with *AND* logic:

* Genre = Jazz AND Composer = Johann Sebastian Bach -> only Jazz albums with that composer

Active filters are reflected in the URL query string so you can bookmark or share filtered views:

----
http://localhost:3005/?genre=Jazz
----

Refreshing the page restores the same filters.

NOTE: Year range filtering (by release year) is not yet available in the current release.
It may be added in a future version.

== Free-Text Search

The *search box* in the header lets you find albums by typing any part of the title, artist, composer, or label.
Search queries are sent to the catalog server as the `q` parameter.

Searched fields:

[cols="1,2", options="header"]
|===
| Field | Example value

| Album title (ALBUM tag)
| "Kind of Blue"

| Album artist (ALBUMARTIST tag)
| "Miles Davis"

| Composer (COMPOSER tag)
| "Johann Sebastian Bach"

| Label (LABEL tag)
| "Blue Note"
|===

Matching is *case-insensitive* and substring-based: typing "blue" matches "Kind of Blue",
"Blue Note", and "Blue Train".

The search is *debounced* - the grid updates 150 ms after you stop typing,
then a new request is sent to the server with the search term.

=== Combining Search and Filters

The search box and the filter panel combine with AND logic.
Typing "miles" while the Jazz genre filter is active shows only Jazz albums that also
contain "miles" in title, artist, composer, or label.

=== Clearing the Search

Click the *x* button to the right of the search box to clear the query.
The grid returns to showing all albums that match the currently active panel filters.

The URL `?q=` parameter is removed when the search is cleared.

== Classical Music Display

kbeatz distinguishes between *popular music* (where the performer is the primary identity)
and *classical music* (where the composer is the primary identity).

When an album has the *COMPOSER* Vorbis tag set, the album card shows:

* *Composer name* as the primary, prominent attribution line.
* *Album artist* (the performing ensemble or conductor) as secondary text.

When the COMPOSER tag is absent, only the ALBUMARTIST is shown as the primary attribution.

Example:

[cols="1,2,2", options="header"]
|===
| Tag | Classical album | Popular album

| ALBUMARTIST
| Berliner Philharmoniker
| Miles Davis

| COMPOSER
| Ludwig van Beethoven
| _(not set)_

| Shown prominently
| Ludwig van Beethoven
| Miles Davis

| Shown as secondary
| Berliner Philharmoniker
| _(not shown)_
|===

== Sort Preferences

The *Sort by* dropdown in the toolbar controls the order in which albums appear in the grid.

[cols="1,3", options="header"]
|===
| Option | Behaviour

| Album Artist (default)
| Albums sorted alphabetically by ALBUMARTIST ascending.

| Composer
| Albums with a COMPOSER tag sorted alphabetically by composer ascending.
  Albums without a COMPOSER tag appear after all composer-attributed albums,
  then sorted by album artist.
|===

The selected sort preference is saved to your browser's `localStorage` and restored
the next time you visit the page.
Clearing your browser data or using a different browser will reset it to "Album Artist".

== Cover Art

Cover art is served by the catalog service from each album's directory.

Resolution order:

1. *Embedded PICTURE block* (FLAC METADATA_BLOCK_PICTURE type 3, front cover)
   extracted directly from one of the album's FLAC files.
2. *folder.jpg* - a JPEG image file named `folder.jpg` in the album directory.
3. *Placeholder* - a grey icon with a musical note symbol is shown in the UI.

If a cover image fails to load after the page has rendered (network error, file removed),
the placeholder is displayed automatically without requiring a page refresh.

Cover art is cached in the browser for 24 hours (`Cache-Control: max-age=86400`).
Forcing a browser refresh (`Ctrl+Shift+R`) bypasses the cache.

== Keyboard Accessibility

All interactive controls are keyboard-accessible:

* *Tab* to navigate between the search box, filter checkboxes, and sort dropdown.
* *Space* or *Enter* to toggle a checkbox.
* *Escape* clears focus from the search box.

Genre, artist, and composer filter groups are labelled with ARIA roles for screen readers.
