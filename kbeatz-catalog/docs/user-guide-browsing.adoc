== Overview

kbeatz is a music collection management platform for FLAC libraries.
The browser UI at `http://localhost:3005` lets you browse your entire album collection
at a glance, narrow it with filters and free-text search, and sort by album artist or composer.

The interface is built on Material Design (MUI). It uses a top App Bar and a navigation
drawer to move between the main areas of the app, and it offers a light and a dark theme that
you can switch at any time. The layout is fully responsive: it adapts from a phone screen up to
a wide desktop monitor without horizontal scrolling.

The reworked UI is described in detail in the platform requirements
(`docs/requirements-ui-rework.md`) and the styling decision is recorded in ADR-013.

== Navigation: App Bar and Drawer

A persistent *App Bar* runs along the top of every page. It contains:

* The app title and the navigation menu (the hamburger button on small screens).
* A *theme toggle* that switches between the light and dark themes (see <<_light_and_dark_theme>>).
* A *language control* that switches between English and German.

The *navigation drawer* lists the main destinations:

[cols="1,2,2", options="header"]
|===
| Destination | Route | Purpose

| Albums
| `/`
| Browse, search, filter, and sort your collection (this guide).

| Library
| `/library`
| Trigger a library scan and watch its progress and any per-album errors.

| Settings
| `/settings`
| Set your default sort preference, language, and theme.
|===

On a desktop (medium width and wider) the drawer is permanent and always visible beside the
content. On a phone or narrow window it collapses to an overlay drawer that you open with the
menu button in the App Bar and dismiss by tapping outside it.

Each destination has its own bookmarkable URL, and the browser Back and Forward buttons move
between them as expected.

== Light and Dark Theme

kbeatz ships a light theme and a dark theme. Use the theme toggle in the App Bar to switch
between them; the change applies instantly with no page reload.

* On your first visit the theme follows your operating system preference
  (`prefers-color-scheme`).
* Once you pick a theme it is saved in your browser and reapplied on every later visit, before
  the page paints, so you never see a flash of the wrong theme.
* You can also set the theme from the Settings page.

Both themes meet WCAG 2.1 AA contrast requirements. If your browser blocks local storage
(for example, private browsing), the theme falls back to your OS preference for that session.

== Album Grid

When the Albums page loads, kbeatz fetches your album summaries from the catalog service and
renders them as a responsive grid of Material cards. The grid is *paginated on the client*: only
one page of cards is shown at a time (see <<_pagination>>).

Each *album card* shows:

* *Cover art* - the front cover image served by the catalog.
  If no cover is available, a placeholder icon is shown instead.
* *Title* - the album (ALBUM Vorbis tag).
* *Primary attribution* - see <<_classical_music_display>>.
* *Year* - the DATE Vorbis tag (year or full ISO date).
* *Genre* chip - the GENRE Vorbis tag.

The grid reflows its column count with the viewport, from a single column on a phone to several
columns on a wide desktop, and never requires horizontal scrolling.

=== Pagination

The album grid shows one page of cards at a time and provides a Material pagination control
below the grid.

* *Page size* has a sensible default (48 cards per page) and can be changed; your choice is
  saved in your browser and reused on later visits.
* *Page controls* let you jump to the next, previous, first, last, or a specific page number.
* The *current page* and *page size* are stored in the URL query string, so a paginated view is
  deep-linkable: you can bookmark or share it, and reloading or using Back/Forward returns you to
  the same page.

----
http://localhost:3005/?page=3&size=48
----

Search and filtering operate over your whole loaded collection, and the paginated view shows the
*filtered* results. Whenever you change the active search or filters, the grid resets to page 1.

When you open an album and then return to the grid, your previous page, scroll position, and
active filters are restored.

=== Scan Progress Banner

If a library scan is in progress when you open the UI, a banner appears at the top:

....
Scanning: 342 / 2000 albums
....

The count updates every 2 seconds.
The banner disappears automatically when the scan completes.
If the scan fails, the banner displays the error message from the server.

== Filtering the Collection

The *filter panel* groups genre, artist, and composer controls. On a desktop it sits beside the
grid; on a phone it is reachable from the App Bar so it does not crowd the cards.
Filters are applied on the client over the full set of loaded album summaries, and the paginated
grid reflects the filtered results. Applying or changing a filter resets the grid to page 1.

=== Genre Filter

Check one or more genre boxes to show only albums matching those genres.
Multiple genres use *OR* logic: checking "Jazz" and "Blues" shows albums in either genre.

=== Artist Filter

Check one or more artist boxes to restrict the grid to albums by those album artists.
Multiple artists use OR logic.

=== Composer Filter

Check one or more composer boxes to show only albums with those composers set.
Albums without a COMPOSER tag are excluded when any composer filter is active.

=== Combining Filters

Different filter axes combine with *AND* logic:

* Genre = Jazz AND Composer = Johann Sebastian Bach -> only Jazz albums with that composer

Active filters are reflected in the URL query string so you can bookmark or share filtered views:

----
http://localhost:3005/?genre=Jazz
----

Refreshing the page restores the same filters.

NOTE: Year range filtering (by release year) is not yet available in the current release.
It may be added in a future version.

== Free-Text Search

The *search box* on the Albums page lets you find albums by typing any part of the title, artist, composer, or label.
The search term is reflected in the URL `q` query parameter so a searched view is deep-linkable.

Searched fields:

[cols="1,2", options="header"]
|===
| Field | Example value

| Album title (ALBUM tag)
| "Kind of Blue"

| Album artist (ALBUMARTIST tag)
| "Miles Davis"

| Composer (COMPOSER tag)
| "Johann Sebastian Bach"

| Label (LABEL tag)
| "Blue Note"
|===

Matching is *case-insensitive* and substring-based: typing "blue" matches "Kind of Blue",
"Blue Note", and "Blue Train".

The search is *debounced* - the grid updates 150 ms after you stop typing.
Searching resets the grid to page 1 of the filtered results.

=== Combining Search and Filters

The search box and the filter panel combine with AND logic.
Typing "miles" while the Jazz genre filter is active shows only Jazz albums that also
contain "miles" in title, artist, composer, or label.

=== Clearing the Search

Click the *x* button to the right of the search box to clear the query.
The grid returns to showing all albums that match the currently active panel filters.

The URL `?q=` parameter is removed when the search is cleared.

== Classical Music Display

kbeatz distinguishes between *popular music* (where the performer is the primary identity)
and *classical music* (where the composer is the primary identity).

When an album has the *COMPOSER* Vorbis tag set, the album card shows:

* *Composer name* as the primary, prominent attribution line.
* *Album artist* (the performing ensemble or conductor) as secondary text.

When the COMPOSER tag is absent, only the ALBUMARTIST is shown as the primary attribution.

Example:

[cols="1,2,2", options="header"]
|===
| Tag | Classical album | Popular album

| ALBUMARTIST
| Berliner Philharmoniker
| Miles Davis

| COMPOSER
| Ludwig van Beethoven
| _(not set)_

| Shown prominently
| Ludwig van Beethoven
| Miles Davis

| Shown as secondary
| Berliner Philharmoniker
| _(not shown)_
|===

== Sort Preferences

The *Sort by* preference controls the order in which albums appear in the grid. Set it on the
*Settings* page (reachable from the navigation drawer).

[cols="1,3", options="header"]
|===
| Option | Behaviour

| Album Artist (default)
| Albums sorted alphabetically by ALBUMARTIST ascending.

| Composer
| Albums with a COMPOSER tag sorted alphabetically by composer ascending.
  Albums without a COMPOSER tag appear after all composer-attributed albums,
  then sorted by album artist.
|===

The selected sort preference is saved to your browser's `localStorage` and restored the next
time you visit. Clearing your browser data or using a different browser resets it to
"Album Artist".

== Settings

The *Settings* page (`/settings`, reachable from the navigation drawer) is the central place for
your personal preferences:

[cols="1,3", options="header"]
|===
| Setting | Behaviour

| Sort preference
| Default album sort order (Album Artist or Composer), as described in <<_sort_preferences>>.

| Language
| Interface language (English or German). The same control is also available in the App Bar.

| Theme
| Light or dark theme. The same toggle is also available in the App Bar (see
  <<_light_and_dark_theme>>).
|===

All of these preferences are stored in your browser. If local storage is unavailable, kbeatz
falls back to in-memory defaults and your operating system theme for that session.

== Cover Art

Cover art is served by the catalog service from each album's directory.

Resolution order:

1. *Embedded PICTURE block* (FLAC METADATA_BLOCK_PICTURE type 3, front cover)
   extracted directly from one of the album's FLAC files.
2. *folder.jpg* - a JPEG image file named `folder.jpg` in the album directory.
3. *Placeholder* - a grey icon with a musical note symbol is shown in the UI.

If a cover image fails to load after the page has rendered (network error, file removed),
the placeholder is displayed automatically without requiring a page refresh.

Cover art is cached in the browser for 24 hours (`Cache-Control: max-age=86400`).
Forcing a browser refresh (`Ctrl+Shift+R`) bypasses the cache.

== Keyboard Accessibility

The reworked UI meets WCAG 2.1 AA in both the light and the dark theme. All interactive
controls are keyboard-accessible with a visible focus indicator:

* *Tab* to navigate between the App Bar controls (theme toggle, language, search box), the
  filter controls, the pagination control, and the album cards.
* *Space* or *Enter* to toggle a checkbox or activate a button.
* *Escape* clears focus from the search box and closes the overlay navigation drawer.

The navigation, filter groups, and form controls carry MUI accessibility roles and labels for
screen readers, and Material transitions respect `prefers-reduced-motion`.
