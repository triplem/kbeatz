== Overview

*kbeatz* is a Kotlin/Ktor + React monorepo skeleton pre-configured for AI-assisted development
with Claude Code.

This documentation covers:

* xref:getting-started.adoc[Getting Started] - first-time setup and project bootstrap
* xref:architecture.adoc[Architecture] - hexagonal structure, API-first design, auth
* xref:developer-guide.adoc[Developer Guide] - module map, hexagonal architecture in practice, end-to-end feature walkthrough, composite Gradle build, domain conventions
* xref:release-process.adoc[Release Process] - how semantic-release creates versions, what gets published, and how to write commits for good release notes
* xref:claude-code-config.adoc[Claude Code Configuration] - skills, rules, hooks, and the SDLC workflow
* xref:claude-process.adoc[Claude Code SDLC Process] - generic AI-assisted SDLC: skills catalog, agents, hooks, memory, quality gates, common patterns
* xref:claude-kbeatz-config.adoc[kbeatz Claude Code Configuration] - kbeatz-specific CLAUDE.md hierarchy, hooks, memory, MCP servers, and recommended workflows
* xref:claude-skills.adoc[Claude Skills Reference] - each skill with trigger condition, expected output, and invocation example
* xref:claude-rules.adoc[Claude Rules Reference] - all rule files with full content summaries
* xref:claude-config.adoc[Claude Configuration Files] - CLAUDE.md hierarchy: global, project root, and per-module
* xref:adding-a-service.adoc[Adding a Service] - step-by-step guide for new backend services
* xref:kbeatz-metadata-pipeline.adoc[Metadata Pipeline Developer Reference] - .kbeatz/ directory layout, KbeatzMetadata JSON schema, multi-disc matching rules, and localPath image convention
* xref:write-lock-reference.adoc[Write-Lock Lifecycle Reference] - .kbeatz-write.lock sentinel: create, delete, startup repair, and failure modes

== Shared Libraries

[cols="1,2"]
|===
|Module |Purpose

|kbeatz-common
|Shared Kotlin library (no port, no deployment): domain exceptions consumed by all modules

|kbeatz-sources
|Metadata library: `MetadataSource`/`MetadataCache` ports; Discogs adapter

|kbeatz-tagger
|FLAC codec + `TaggerService` + id-file parser; library consumed by catalog and CLI
|===

== Technology Stack

[cols="1,2"]
|===
|Layer |Technology

|Backend language
|Kotlin 2.x + JVM 21

|Backend framework
|Ktor 3.x (CIO engine)

|Build system
|Gradle 9 with Kotlin DSL + composite builds

|Persistence
|H2 (embedded JVM database, v1) + Exposed ORM + HikariCP + Liquibase. PostgreSQL: migration target in v2 (see ADR-006).

|Authentication
|None in v1 (trusted LAN). Keycloak JWT/OIDC planned for Phase 3+ (not implemented in v1 or v2).

|Frontend
|React 19 + TypeScript + Vite + Axios

|API contract
|OpenAPI 3.1 (spec-first, code-generated on both sides)

|Code quality
|Detekt + Kover (≥ 80% coverage) + ESLint

|Observability
|OpenTelemetry (traces via gRPC to OTEL collector)

|Documentation
|AsciiDoc + Asciidoctor Gradle plugin
|===
